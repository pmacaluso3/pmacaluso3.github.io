Sticking things together
4/2/2015
...or, using arrays and hashes in Ruby
(...back to index)

You've got lists, I've got lists. It's so natural to make lists that I'd bet they've been around as long as people have had things to... well, list. If you've got some best friends, you can make a best friends list. If you've got features you want in your next car, you can make a features list. In Ruby, arrays and hashes are two different ways to make a list. Both of these techniques allow you not only to organize data, but also to manipulate it (add new items, change one current item into something else, etc). I want to start by telling you about arrays.

Sooner or later, you're going to have to go get groceries. We're going to make an array that contains your shopping list for you. That would look like this:

shopping_list = ["bread","milk","eggs"]

A few things might immediately raise your eyebrows:

  1. Why is there an underscore in shopping_list rather than a space?
  2. I've been buying groceries for years and never needed to use an equal sign. Why is it there?
  3. What are the square brackets for?
  4. Do I need quotation marks around my groceries?

Answered in order:

  1. Ruby uses a space to break up two thoughts in its brain. If you say "shopping list", it thinks that "shopping" and "list" are two separate things. You have to use something other than a space to convince it that they're the same thing. Remember, Ruby doesn't speak English in the same way you and I do.
  2. An equal sign is how we tell Ruby "I want you to remember to associate this information with this label." In this case, we want it to remember to associate the items with the label shopping_list. The label always has to go on the left of the equal sign, and the information goes on the right. We call this "assigning" the information to the label.
  3. The square brackets simply tell Ruby that these things will be grouped together into an array. Remember, an array is just another word for list.
  4. Ruby only recognizes textual data if it's enclosed in quotation marks. Otherwise, it thinks it's a label that stands for something else. See how shopping_list wasn't enclosed in quotes? Because of this, Ruby knows it's a label rather than just some regular text.

Now you've got your shopping list stored in an array! Arrays can do cool things, like referring to each item in the list by number. Imagine asking the question, "What's the (first/second/etc) item on my list?" We would call the number of the item you're looking for (whether it's first, second, or whatever) the index of that item. In Ruby, we can ask an array to spit out the item stored at a particular index like this:

shopping_list[index]

except that you should use a number instead of the word "index". So in our list of items above, which would be item #1 (in other words, which item would you get if you typed shopping_list[1]?)

Go on, look back, think about it.

This was kinda a trick question, but the answer isn't unreasonable. Item #1 in our list is not "bread", but rather it's "milk". The very first item in a list gets assigned index 0 rather than one. So, "bread" is #0, and "eggs" is #2.

There will be some situations in which you'll want to refer to the items in your list by some other indicator other than their index numbers. For example, if you wanted to put your groceries in order from least expensive to most expensive, you'd need to refer to them by their prices rather than their indices. To do a trick like this, you'll need to use a hash rather than an array. A hash is a lot like an array, except that in a hash you can assign each item a reference value other that can be different from it's position number. If our grocery items had the prices listed below on the left, the way to store this information in a hash is given on the right.

Grocery Prices
Bread 2.99
Milk 1.89
Eggs 1.19
shopping_list = {2.99 => "Bread", 1.89 => "Milk", 1.19 => "Eggs"}

Here's what's going on in that code:

  • The curly braces simply tell Ruby that you're making a hash as opposed to an array.
  • The => symbols create pairs of items that go together. We call these "key-value pairs". The first entry of each pair is the key and the second one is the value. In this case, the key of "Bread" corresponds to the value of 2.99, and so on.

To complete our trick of referring to items by something other than their indices (remember that here, we want to refer to items by price so that we can sort them in order of expensiveness), we ask the hash for the item with any particular price. Let's ask the hash, "Which item costs 1.19?"

shopping_list[1.19]

This code will spit out the item that is associated with the key of 1.19, namely, "Eggs". Then, you could go on to ask Ruby to tell you the item that costs 1.89, and finally, the item that costs 2.99. Think about how you would write that, and check the answer at the end of this post.

Arrays and hashes can both be used to group items together into a list. The main difference between the two is that in an array, items are referred to by their place in the list, or index. Don't forget that the first item gets the #0 instead of #1, which can be confusing at first. In a hash, values can be referred to not by their index, but by a key that was assigned arbitrarily when the hash was made. This lets you ask for items according to some criteria other than their position in the list.

The answer to how you can ask a hash to give you its values associated with the keys of 1.89 and 2.99:

shopping_list[1.89]

shopping_list[2.99]
